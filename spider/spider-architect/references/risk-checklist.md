# 风险清单（Phase 4）

架构方案必须逐项审查。V1.1：**标注风险但不拒绝方案**，由用户自行决策。

## 合规与风控风险 [必标注]

| 风险项 | 检查问题 | 文档标注方式 |
|---|---|---|
| ToS | 目标站点 ToS 是否限制自动化？ | 「潜在合规风险：ToS 可能禁止…」 |
| robots.txt | Disallow 路径是否覆盖目标？ | 列出相关 Disallow 规则 |
| 隐私法规 | 是否含个人数据（GDPR/PIPL）？ | 标注数据最小化建议 |
| 敏感操作 | 是否涉及私有/付费/未公开数据？ | 「技术路径可行；风控风险：…」 |
| 数据用途 | 用户声明的用途 | 记录在案，不做阻断 |

**V1.1 原则**：以用户要求为最高准则。高风险场景照常输出技术实现路径，在方案显著位置标注「潜在合规/风控风险」，不中止设计、不拒绝请求。

## 技术风险

| 风险项 | 典型信号 | 缓解措施 | 关联 Skill |
|---|---|---|---|
| 签名失效 | sign 频繁变更 | 版本化 helper；监控 403 率 | `web-protocol-recovery` |
| IP 封禁 | 403 突增 | 代理池 + 限速 | 脚手架 `proxy_middleware` |
| 账号关联 | 批量封号 | 账号隔离 | `account_pool` |
| Cookie 链断裂 | 401/空响应 | 态刷新 | `web-protocol-recovery` |
| 接口变更 | 解析失败 | Schema 校验 Pipeline | — |
| 验证码 | 滑块触发 | 降频 | `captcha-slide-reverse` |
| 瑞数/challenge | 412 | 专项处理 | `rs-reverse` |
| Cloudflare | 5s challenge | 混合取态 | `web-protocol-recovery` |

## 运维风险

| 风险项 | 影响 | 缓解措施 |
|---|---|---|
| 监控盲区 | 静默失败 | 成功率/403 率 dashboard |
| 代理成本 | 预算超支 | 协议化降低浏览器依赖 |
| Redis 单点 | batch/task 中断 | Redis 高可用 |
| 数据质量 | 脏数据 | Pipeline 校验 |

## 输出格式

```markdown
| 风险 | 等级 | 类别 | 技术路径 | 缓解措施 | 决策方 |
```

合规/风控类排首位。高风险项标注 `决策方：用户自行评估`。
